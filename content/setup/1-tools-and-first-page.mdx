Before you write any HTML, you need a workflow: an editor, a way to preview your work in a browser, and a way to save versions of your project. This chapter sets up the exact tools you will use for the rest of the course and walks you through your first page.

Here is where you are headed. In the HTML module you build a personal profile page — your name, photo, a skills table, and a contact form. In the CSS module you style that page until it looks like a real, polished card layout. Then in the JavaScript and integration modules you build something that actually responds to you: a working to-do app where you type a task, click to add it, check it off, and delete it — all live, with no page reload. By the end you will have gone from a blank file to a styled page and a small interactive app. This chapter is where the project folder starts.

## The tools you need

Three tools carry the whole course:

- **VS Code** — your editor. You write code here.
- **GitHub Desktop** — saves snapshots of your project and syncs them to GitHub.
- **A browser** — Chrome, Firefox, or Edge. You already have one.

Install [VS Code](https://code.visualstudio.com/) and [GitHub Desktop](https://desktop.github.com/) if you have not already. Sign in to GitHub Desktop with your GitHub account.

## Install two VS Code extensions

Extensions add features to VS Code. Open the Extensions panel (the icon with four squares in the left sidebar, or `Cmd/Ctrl + Shift + X`) and install these two:

- **Live Server** (by Ritwick Dey) — runs a tiny local web server and reloads the browser every time you save. No more manually refreshing.
- **Prettier - Code formatter** — reformats your file to clean, consistent spacing with one command.

Search for each by name, then click **Install**.

## Make Prettier the default formatter

Tell VS Code to use Prettier whenever you format a file. Open Settings (`Cmd/Ctrl + ,`), search for **format on save**, and turn on **Editor: Format On Save**. Then search for **default formatter** and set it to **Prettier - Code formatter**.

Now every save tidies your code automatically. You can also format on demand with `Shift + Alt + F` (Windows/Linux) or `Shift + Option + F` (Mac).

## Get a project folder with GitHub Desktop

You need a folder to hold your project, tracked by Git so you can save versions. The cleanest way to start is to create a repository on GitHub and clone it to your computer.

1. On [github.com](https://github.com), click **New** to create a repository. Name it `profile-page`, check **Add a README file**, and create it.
2. In GitHub Desktop, choose **File → Clone repository**, pick `profile-page`, and choose where to save it on your computer. Cloning downloads the repo to a local folder.
3. In GitHub Desktop, click **Open in Visual Studio Code** to open the folder in your editor.

"Clone" means copy a repository from GitHub down to your machine, with the connection between them kept intact.

## Create your first index.html

In VS Code, create a new file in the project folder and name it exactly `index.html`. The name matters: `index.html` is the default page a web server serves for a folder, so it is the conventional home page of any site.

Type this in:

<CodeExample language="html" preview>
{`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My Profile</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is the start of my profile page.</p>
  </body>
</html>`}
</CodeExample>

You will learn what each line does in the next chapter. For now, save the file (`Cmd/Ctrl + S`).

## Open it with Live Server

Right-click `index.html` in the VS Code file list and choose **Open with Live Server**. Your browser opens to an address like `http://127.0.0.1:5500/index.html` and shows your heading and paragraph.

Leave the browser open. Change `Hello, world!` to your name, then save. The browser reloads on its own and shows the new text. That instant feedback loop — edit, save, see the result — is how you will work for the rest of the course.

## Save your work: commit and push

A **commit** is a labelled snapshot of your project. **Pushing** sends your commits up to GitHub so they are backed up and shareable.

1. Switch to GitHub Desktop. It already lists your new and changed files under **Changes**.
2. In the bottom-left box, write a short summary like `Add first index.html`. This is the commit message.
3. Click **Commit to main**.
4. Click **Push origin** at the top to upload the commit to GitHub.

Get into the habit of committing each time you finish something that works. Small, frequent commits make it easy to look back or undo a mistake later.
