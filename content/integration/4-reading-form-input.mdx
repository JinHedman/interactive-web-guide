Events tell you *that* the user did something. Now you need to read *what* they entered, so your app can act on it. A to-do app has to know the text of the new task. A real form has to know which checkboxes are ticked and which option is chosen. This chapter is about pulling values out of form controls and giving the user feedback right on the page.

## Read a text input's value

A text input's current text lives in its `value` property. Read it inside an event handler to find out what the user typed.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <input id="taskInput" placeholder="Type a task...">
    <button id="readBtn">Read it</button>
    <p id="out"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const input = document.getElementById("taskInput");
const readBtn = document.getElementById("readBtn");
const out = document.getElementById("out");

readBtn.addEventListener("click", function () {
  out.textContent = "The input holds: " + input.value;
});`
  }
]} />

`value` is always a string — even for a number input. To clear a field, set `value` back to `""`.

## Read checkbox and radio state

Checkboxes and radio buttons do not use `value` to tell you if they are selected. They use a boolean property called `checked`: `true` when ticked, `false` when not.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <label>
      <input type="checkbox" id="urgent"> Mark as urgent
    </label>
    <button id="checkBtn">Check status</button>
    <p id="out"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const urgent = document.getElementById("urgent");
const checkBtn = document.getElementById("checkBtn");
const out = document.getElementById("out");

checkBtn.addEventListener("click", function () {
  if (urgent.checked) {
    out.textContent = "This task is urgent.";
  } else {
    out.textContent = "This task is normal priority.";
  }
});`
  }
]} />

Tick the box, then click the button. Radio buttons work the same way: you check `.checked` on each one to find which is selected.

## Read a select value

A `<select>` dropdown exposes the chosen option through its `value` property — the `value` attribute of the selected `<option>`.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <label for="priority">Priority:</label>
    <select id="priority">
      <option value="low">Low</option>
      <option value="medium">Medium</option>
      <option value="high">High</option>
    </select>
    <p id="out"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const priority = document.getElementById("priority");
const out = document.getElementById("out");

priority.addEventListener("change", function () {
  out.textContent = "Priority set to: " + priority.value;
});`
  }
]} />

The `change` event fires when the user picks a different option. Then `priority.value` gives you `"low"`, `"medium"`, or `"high"`.

## Validate and show feedback in the DOM

Before you act on input, check it. If a user submits an empty task, you should tell them — but not with `alert()`, which is jarring and blocks the page. Instead, write a message into an element on the page. This is friendlier and lets you style it.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <form id="taskForm">
      <input id="taskInput" placeholder="New task">
      <button type="submit">Add</button>
    </form>
    <p id="message"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `.error { color: crimson; }
.ok { color: green; }`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const form = document.getElementById("taskForm");
const input = document.getElementById("taskInput");
const message = document.getElementById("message");

form.addEventListener("submit", function (event) {
  event.preventDefault();
  const text = input.value.trim();

  if (text === "") {
    message.textContent = "Please type a task before adding.";
    message.className = "error";
    return;
  }

  message.textContent = "Added: " + text;
  message.className = "ok";
  input.value = "";
});`
  }
]} />

Two details worth noticing. `input.value.trim()` removes leading and trailing spaces, so a field of only spaces counts as empty. And `return` ends the handler early when validation fails, so the "added" code below it never runs. Swapping the `className` between `error` and `ok` lets your CSS color the message appropriately.
