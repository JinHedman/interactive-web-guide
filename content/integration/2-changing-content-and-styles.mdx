<InShort>Once you hold an element, you change it through `textContent` (text), `classList` (classes), `style` (inline CSS), and `getAttribute`/`setAttribute` (attributes) — and the page updates instantly.</InShort>

Selecting an element is only useful if you can then change it. Once you hold an element in a variable, you can rewrite its text, switch its classes, tweak its styles, and edit its attributes — and the page updates instantly. This is how a to-do item gets crossed off, or a status message appears. This chapter covers the four ways you will change elements every day.

## Change text with textContent

Every element has a `textContent` property. Read it to get the text inside, or assign to it to replace that text.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <h1 id="appTitle">Loading...</h1>
    <p id="count">0 tasks</p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const title = document.getElementById("appTitle");
const count = document.getElementById("count");

title.textContent = "My Tasks";
count.textContent = "3 tasks";`
  }
]} />

Assigning to `textContent` replaces everything inside the element with plain text. It treats the value as text, not HTML — so `"<b>hi</b>"` would show those angle brackets literally, never bold text. There is a sibling property, `innerHTML`, that *does* parse its value as HTML — and that is exactly why `textContent` is safer for anything a user typed: with `innerHTML`, a malicious string like `"<script>...</script>"` could inject markup or scripts into your page, while `textContent` always shows it as harmless text. Reach for `textContent` for user-provided data.

## Toggle and swap classes with classList

You already wrote CSS classes. From JavaScript, `classList` lets you add, remove, and toggle those classes on an element — which is the cleanest way to change how something looks.

- `classList.add("done")` — adds the class.
- `classList.remove("done")` — removes it.
- `classList.toggle("done")` — adds it if missing, removes it if present.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <ul>
      <li id="task1">Buy milk</li>
      <li id="task2">Walk the dog</li>
    </ul>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `.done { text-decoration: line-through; color: gray; }`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const task1 = document.getElementById("task1");
const task2 = document.getElementById("task2");

task1.classList.add("done");
task2.classList.toggle("done");
task2.classList.toggle("done");`
  }
]} />

The first task gets the `done` class and shows the strike-through. The second is toggled on, then off, so it looks unchanged. `toggle` is perfect for a checkbox-style behavior: click once to complete, click again to un-complete.

Two related forms are worth knowing. Pass `toggle` a second argument — a boolean — and it forces the class **on** when that value is `true` and **off** when it is `false`, instead of flipping. That lets you drive a class straight from a condition. And `classList.contains("done")` returns `true`/`false` so you can *check* whether a class is currently present.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <li id="task">Walk the dog</li>
    <p id="state"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `.done { text-decoration: line-through; color: gray; }`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const task = document.getElementById("task");
const state = document.getElementById("state");

const isDone = true;
task.classList.toggle("done", isDone); // force on because isDone is true

state.textContent =
  "Has done class? " + task.classList.contains("done"); // true`
  }
]} />

Because `isDone` is `true`, the class is forced on; `contains` then confirms it is there. The two-argument `toggle` is ideal when you already hold a boolean (like a task's completed state) and want the class to match it.

## Set inline styles with the style property

You can also change a single CSS property directly through the `style` property. CSS property names that contain a hyphen become camelCase: `background-color` becomes `backgroundColor`.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <p id="banner">Important task!</p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const banner = document.getElementById("banner");
banner.style.color = "white";
banner.style.backgroundColor = "crimson";
banner.style.padding = "0.75rem";`
  }
]} />

This works, but each line sets one property in one place. For anything beyond a quick tweak, a class is cleaner.

## Read and set attributes

Beyond text and styles, elements have attributes — `src`, `href`, `disabled`, `data-*`, and so on. Read one with `getAttribute` and change one with `setAttribute`.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <a id="link" href="/old">A link</a>
    <p id="report"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const link = document.getElementById("link");
const report = document.getElementById("report");

const oldHref = link.getAttribute("href");
link.setAttribute("href", "/new");

report.textContent =
  "Changed href from " + oldHref + " to " + link.getAttribute("href");`
  }
]} />

`setAttribute` takes two arguments: the attribute name and its new value. `getAttribute` takes just the name and returns the current value.

## Stash your own data with data-* attributes

<InShort>Custom `data-*` attributes let an element carry your own data — most often *which* item it represents — and you read them through the `dataset` property (`data-id` becomes `element.dataset.id`).</InShort>

HTML lets you invent your own attributes as long as they start with `data-`, like `data-id="42"`. They are a clean place to attach information an element needs — most often *which* item of your data this element represents. You could read them with `getAttribute("data-id")`, but JavaScript gives you a tidier path: the `dataset` property. A `data-id` attribute shows up as `element.dataset.id` (the `data-` prefix is dropped).

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <li id="task" data-id="42" data-priority="high">Walk the dog</li>
    <p id="report"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const task = document.getElementById("task");
const report = document.getElementById("report");

console.log(task.dataset.id);       // "42"
console.log(task.dataset.priority); // "high"

// you can also set one — it updates the data-* attribute
task.dataset.id = "99";

report.textContent = "This element's id is " + task.dataset.id;`
  }
]} />

Reading `task.dataset.id` returns the string `"42"`, and `task.dataset.priority` returns `"high"`. Values always come back as strings. This is the standard way to remember which row a button belongs to — handy once a single handler deals with many items.

## Prefer classes over inline styles

When you want to change appearance, reach for `classList` first. A class keeps all the styling rules in your CSS file, where they belong, and lets one toggle switch many properties at once. Inline styles scatter design decisions through your JavaScript and are harder to override. Use `style` for one-off, dynamic values (like a width computed at runtime); use classes for everything that has a name and a meaning, like `done` or `error`.
