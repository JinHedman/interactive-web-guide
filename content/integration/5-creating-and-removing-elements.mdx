<InShort>`createElement` builds a new element, `append` adds it to the page, and `remove` takes one off — the building blocks for a UI that grows and shrinks at runtime.</InShort>

Until now you have changed elements that already existed in your HTML. But a to-do app cannot know its tasks ahead of time — the user adds them as they go. So you need to **create** new elements from JavaScript and add them to the page, and **remove** them when they are no longer wanted. This chapter gives you those final building blocks. After this, you have everything you need for the capstone.

## Create an element

`document.createElement` makes a fresh element of the tag you name. It exists only in memory at first — it is not on the page until you add it.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <ul id="taskList"></ul>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const list = document.getElementById("taskList");

const li = document.createElement("li");
li.textContent = "Buy milk";
list.append(li);`
  }
]} />

Three steps: create the element, set its content, and add it to the page. The list started empty in the HTML; the script filled it in.

## Add elements with append

`append` adds your new element as the last child of a parent. (You will also see `appendChild`, the older method — it does the same job for a single element. `append` is newer and a little more flexible, so prefer it.) One handy bit of that flexibility: `append` can take several nodes at once — `parent.append(a, b, c)` adds them all in order — which the capstone uses to drop a checkbox, a label, and a delete button into one list item in a single call.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <ul id="taskList">
      <li>Existing task</li>
    </ul>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const list = document.getElementById("taskList");

const first = document.createElement("li");
first.textContent = "Walk the dog";
list.append(first);

const second = document.createElement("li");
second.textContent = "Write some code";
list.append(second);`
  }
]} />

Each `append` puts the new item at the end, so they appear in the order you added them, after the task that was already there.

## Remove an element

Any element you hold in a variable can remove itself from the page with `remove`. No parent reference needed.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <ul id="taskList">
      <li id="keep">Keep this one</li>
      <li id="drop">Remove this one</li>
    </ul>
    <button id="delBtn">Delete the second task</button>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const drop = document.getElementById("drop");
const delBtn = document.getElementById("delBtn");

delBtn.addEventListener("click", function () {
  drop.remove();
});`
  }
]} />

Click the button and the second item disappears. In a to-do app, each task will carry its own delete button whose click handler calls `remove` on that task.

## Render a list from an array

<InShort>Keep your data in an **array** and rebuild the matching elements from it whenever it changes — the array is the source of truth, and this "render from the array" pattern is the engine of the capstone.</InShort>

Here is the pattern your capstone is built on. Keep your data in an **array**, then loop over it to build the matching elements. When the data changes, you clear the list and rebuild it — so what you see always reflects the data. Notice the shape of the data: each task is an **object** like `{ text: "Buy milk" }`, not a bare string. That gives each task room to grow more fields later (the capstone adds a `done` flag), so we use that shape from the start.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <ul id="taskList"></ul>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const list = document.getElementById("taskList");
const tasks = [
  { text: "Buy milk" },
  { text: "Walk the dog" },
  { text: "Write some code" },
];

function render() {
  list.textContent = "";
  for (const task of tasks) {
    const li = document.createElement("li");
    li.textContent = task.text;
    list.append(li);
  }
}

render();`
  }
]} />

Read the `render` function closely. It first empties the list with `list.textContent = ""`, then loops over `tasks` and creates one `<li>` per item, reading the text out of each object with `task.text`. Call `render` again any time `tasks` changes and the page redraws itself from the current data. This "data in an array, render from the array" idea is the whole engine of the capstone — and, as you will see at the end of the module, the core idea behind modern frameworks too.

    <title>To-Do List</title>
  </head>
  <body>
    <h1 id="appTitle">My Tasks</h1>
    <form id="taskForm">
      <input id="taskInput" placeholder="What needs doing?">

      name: "script.js",
      lang: "js",
      code: `const form = document.getElementById("taskForm");
const input = document.getElementById("taskInput");
const list = document.getElementById("taskList");

const tasks = [
  { text: "Buy milk" },
  { text: "Walk the dog" },
];

function render() {
  list.textContent = "";
  for (const task of tasks) {
    const li = document.createElement("li");
    li.textContent = task.text;
    list.append(li);
  }
}

form.addEventListener("submit", function (event) {
  event.preventDefault();
  const text = input.value.trim();
  if (text === "") {
    return;
  }
  tasks.push({ text: text });
  input.value = "";
  render();
});

render();`
    }
  ]}
/>
