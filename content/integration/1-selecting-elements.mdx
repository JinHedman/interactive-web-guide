You know the DOM is a live tree the browser builds from your HTML. Now you will reach into that tree from JavaScript. Before you can change anything on a page, you have to *find* the element you want. This chapter is about selecting elements — the first step in every piece of interactivity you will ever write.

Through this module you will build toward one project: a working to-do list. We start by learning how to grab the pieces of its page.

## Select one element by id

The most direct way to find an element is by its `id`. Give an element an `id` in your HTML, then call `document.getElementById` with that exact id string.

<CodeExample language="html" preview>
{`<!doctype html>
<html lang="en">
  <body>
    <h1 id="appTitle">My To-Do List</h1>
    <script>
      const title = document.getElementById("appTitle");
      title.textContent = "Selected from JavaScript!";
    </script>
  </body>
</html>`}
</CodeExample>

The script ran after the page loaded, found the `<h1>`, and changed its text. Notice `getElementById` takes the id *without* a `#` in front of it — just the plain string `"appTitle"`.

## Select with querySelector

`getElementById` only works for ids. `querySelector` is more flexible: you pass it a **CSS selector** — the same kind you wrote in the CSS module — and it returns the **first** matching element.

<CodeExample language="html" preview>
{`<!doctype html>
<html lang="en">
  <body>
    <h1 class="title">My To-Do List</h1>
    <p class="note">Pick the first matching element.</p>
    <script>
      const heading = document.querySelector(".title");
      const para = document.querySelector("p");
      heading.textContent = "Found by class selector";
      para.textContent = "Found by element selector";
    </script>
  </body>
</html>`}
</CodeExample>

Because it takes a CSS selector, you write `".title"` with the dot for a class, `"#appTitle"` with the hash for an id, and `"p"` for a tag. If nothing matches, `querySelector` returns `null`.

## Select many with querySelectorAll

`querySelector` stops at the first match. To get **every** matching element, use `querySelectorAll`. It returns a `NodeList` — a list you can loop over.

<CodeExample language="html" preview>
{`<!doctype html>
<html lang="en">
  <body>
    <ul>
      <li>Buy milk</li>
      <li>Walk the dog</li>
      <li>Write some code</li>
    </ul>
    <script>
      const items = document.querySelectorAll("li");
      items.forEach((item) => {
        item.textContent = item.textContent.toUpperCase();
      });
    </script>
  </body>
</html>`}
</CodeExample>

`querySelectorAll` always returns a list, even if only one element matches (or none). You loop over it with `forEach`, just like an array. This is the same `forEach` from the loops chapter, now written with the arrow-function form `(item) => { ... }` from the functions chapter. Each `item` in the loop is one element you can work with.

## Store the result and check for null

Selecting an element gives you back a value. Store it in a `const` so you can use it more than once. But remember: if the selector matches nothing, you get `null` — and trying to use `null` throws an error.

<CodeExample language="html" preview>
{`<!doctype html>
<html lang="en">
  <body>
    <p id="status">Checking...</p>
    <script>
      const status = document.getElementById("status");
      const missing = document.getElementById("doesNotExist");

      if (missing === null) {
        status.textContent = "The element 'doesNotExist' was not found.";
      } else {
        status.textContent = "Found it.";
      }
    </script>
  </body>
</html>`}
</CodeExample>

In real apps you guard against missing elements, because a typo in a selector or a script that runs before the element exists both produce `null`, and trying to use `null` throws an error. The elements we select in the rest of this module are known to exist in the page we wrote, so we keep it simple and skip the check — but it is good to know the value you get back when nothing matches.
