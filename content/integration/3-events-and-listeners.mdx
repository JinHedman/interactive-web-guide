So far your scripts run once, top to bottom, the moment the page loads. Real apps react to the user: a click, a keystroke, a form submission. An **event** is the browser's way of announcing that something happened, and an **event listener** is the function you ask the browser to run when it does. This is the heart of interactivity — and the moment your to-do list starts to feel alive.

## Listen for a click

Call `addEventListener` on an element. You give it two things: the name of the event (a string like `"click"`) and a function to run when the event fires. That function is called a **handler** or **callback**.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <button id="addBtn">Add task</button>
    <p id="status">No clicks yet.</p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const addBtn = document.getElementById("addBtn");
const status = document.getElementById("status");
let clicks = 0;

addBtn.addEventListener("click", function () {
  clicks = clicks + 1;
  status.textContent = "You clicked " + clicks + " time(s).";
});`
  }
]} />

Click the button in the preview. Each click runs the handler again, so the count goes up. The script set up the listener once; the browser calls your function every time the event happens after that.

The handler here is written as a full `function () { ... }` for readability. An arrow function — `() => { ... }` — works identically as a handler, and you will see both styles across this module; pick whichever reads more clearly.

## Listen for typing with input

The `input` event fires every time the value of an input field changes — on every keystroke. It is how you respond to a user as they type.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <input id="taskInput" placeholder="Type a task...">
    <p id="echo">You typed nothing yet.</p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const input = document.getElementById("taskInput");
const echo = document.getElementById("echo");

input.addEventListener("input", function () {
  echo.textContent = "You typed: " + input.value;
});`
  }
]} />

Type in the box and watch the paragraph update live. You read what the user typed with `input.value` — the next chapter goes deeper on reading input.

## The event object

Your handler can take one argument: the **event object**. The browser passes it in automatically, and it carries details about what happened — including `event.target`, the element the event came from.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <button id="btn">Inspect this event</button>
    <p id="out"></p>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const btn = document.getElementById("btn");
const out = document.getElementById("out");

btn.addEventListener("click", function (event) {
  out.textContent =
    "Event type: " + event.type +
    " | from element: " + event.target.tagName;
});`
  }
]} />

Here `event.type` is `"click"` and `event.target.tagName` is `"BUTTON"`. The event object becomes especially useful later, when one listener handles clicks on many items and you need to know which one was clicked.

## Handle a form submit and stop the reload

When a user submits a `<form>` — by clicking a submit button or pressing Enter — the browser fires a `submit` event and then, by default, **reloads the page** to send the data somewhere. For an app that handles things in JavaScript, that reload wipes your work. Call `event.preventDefault()` to stop it.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
  </head>
  <body>
    <form id="taskForm">
      <input id="taskInput" placeholder="New task">
      <button type="submit">Add</button>
    </form>
    <ul id="taskList"></ul>
    <script src="script.js"></script>
  </body>
</html>`
  },
  {
    name: "script.js",
    lang: "js",
    code: `const form = document.getElementById("taskForm");
const input = document.getElementById("taskInput");
const list = document.getElementById("taskList");

form.addEventListener("submit", function (event) {
  event.preventDefault();
  const li = document.createElement("li");
  li.textContent = input.value;
  list.append(li);
  input.value = "";
});`
  }
]} />

Type a task and press Enter or click Add. Without `event.preventDefault()`, the page would reload and the list would vanish. With it, you stay on the page and your JavaScript runs. A `<form>` with a submit handler is the right way to capture "the user finished entering something" — it works with both the button and the Enter key. (You will meet `createElement` properly in chapter 5; here it is just a preview of where this is heading.)
