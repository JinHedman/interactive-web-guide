Most of a web page is text, and HTML gives you specific elements to describe what each piece of text *is*: a heading, a paragraph, a list item. Choosing the right element matters — it shapes the page outline, helps screen readers, and makes your markup readable. In this chapter you add real written content to your profile page.

## Headings: h1 through h6

Headings label sections of a page. There are six levels, from `<h1>` (most important) down to `<h6>` (least). They create the page's outline, like the headings in a document.

<CodeExample language="html" preview>
{`<h1>Ada Lovelace</h1>
<h2>About me</h2>
<h3>Background</h3>
<h2>Projects</h2>`}
</CodeExample>

Two rules:

- Use **one `<h1>`** per page — it is the page's main title.
- **Do not skip levels.** Go `<h1>` then `<h2>`, not `<h1>` straight to `<h3>`. The levels describe structure, not size; if a heading looks too big, change it with CSS later, do not reach for a lower level to shrink it.

## Paragraphs

Body text goes in `<p>` elements. Each paragraph is its own `<p>`. The browser adds space between them automatically.

<CodeExample language="html" preview>
{`<h2>About me</h2>
<p>I started out fascinated by patterns in numbers.</p>
<p>These days I am learning to build for the web, one element at a time.</p>`}
</CodeExample>

Do not try to make two paragraphs by putting blank lines inside one `<p>` — the browser collapses whitespace. Use a separate `<p>` for each.

## Lists

There are two everyday list types:

- An **unordered list** (`<ul>`) for items where order does not matter — it renders with bullets.
- An **ordered list** (`<ol>`) for steps or rankings — it renders with numbers.

Both hold list items, each in an `<li>`.

<CodeExample language="html" preview>
{`<h2>Skills I'm learning</h2>
<ul>
  <li>HTML structure and semantics</li>
  <li>CSS layout and design</li>
  <li>JavaScript interactivity</li>
</ul>

<h2>My learning plan</h2>
<ol>
  <li>Finish the HTML module</li>
  <li>Style the page with CSS</li>
  <li>Add interactivity with JavaScript</li>
</ol>`}
</CodeExample>

The browser handles the bullets and numbers — you never type them yourself.

## Emphasis: em and strong

To stress a word, use `<em>` (emphasis, shown italic) or `<strong>` (strong importance, shown bold).

<CodeExample language="html" preview>
{`<p>This deadline is <strong>today</strong>, so please <em>do not</em> wait.</p>`}
</CodeExample>

You may have seen `<i>` and `<b>` for italic and bold. Prefer `<em>` and `<strong>`: they carry **meaning** (emphasis and importance) that screen readers can convey, while `<i>` and `<b>` are purely visual. Same look, more meaning — that is the HTML way.

## br and hr, used sparingly

Two single-tag elements show up occasionally:

- `<br>` forces a line break within text. Use it only where the break is part of the content, like lines of a poem or an address — not to add spacing.
- `<hr>` marks a thematic break between sections, rendered as a horizontal rule.

<CodeExample language="html" preview>
{`<p>Ada Lovelace<br>London, England</p>
<hr>
<p>Next section starts here.</p>`}
</CodeExample>

If you find yourself stacking `<br>` tags to push things around, that is a job for CSS spacing, which comes later.
