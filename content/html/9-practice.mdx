<InShort>Fix-this debugging drills plus a few elements the core lessons skipped (`<dl>`, `<details>`, table `scope`/`colspan`) — built on fresh content and validated before you call each done.</InShort>

Extra reps, on content you have not seen before. These drills mix fix-this debugging with a handful of useful elements beyond the core lessons. Build each in a plain `.html` file, preview with Live Server, and validate at [validator.w3.org](https://validator.w3.org/) before you call it done.

<Exercise
  title="Fix the broken skeleton"
  goal="Find and fix five planted bugs in a small HTML document so it becomes valid."
  steps={[
    "In VS Code, create a file named broken.html and paste the broken markup from the expected result below (the version with the bugs).",
    "Bug 1: the file is missing its DOCTYPE on line 1 — add it.",
    "Bug 2: the title sits inside the body — move it into the head where metadata belongs.",
    "Bug 3: the charset meta is written with a closing tag — meta is an empty element, so remove the </meta>.",
    "Bug 4: a paragraph is never closed — add the missing </p>.",
    "Bug 5: the strong and em tags close in the wrong order — fix the nesting so tags close in reverse order.",
    "Paste the result into validator.w3.org, confirm zero errors, then commit in GitHub Desktop with the message 'Fix broken HTML skeleton'."
  ]}
  expected="A document that passes the W3C validator with no errors. The starting (broken) version looks like this — your job is to repair it:"
  solution={`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Tuesday Coding Meetup</title>
  </head>
  <body>
    <h1>Tuesday Coding Meetup</h1>
    <p>Doors open at <strong>6 <em>sharp</em></strong>.</p>
    <p>Bring a laptop and a project you are stuck on.</p>
  </body>
</html>`}
/>

<Exercise
  title="De-div-ify a page"
  goal="Replace div soup with real semantic landmarks, and leave the one div that is genuinely correct alone."
  steps={[
    "Create landmarks.html and start from the div-only version shown in the expected result below.",
    "Swap div class=\"header\" for a header element.",
    "Swap div class=\"nav\" for a nav element.",
    "Swap div class=\"main\" for a main element (there must be exactly one per page).",
    "Swap div class=\"footer\" for a footer element.",
    "Leave the inner grouping div alone — it groups two inline pieces with no semantic meaning, which is exactly what a div is for.",
    "Validate, then commit with the message 'Replace divs with semantic landmarks'."
  ]}
  expected="A page built from header, nav, main, and footer, with one justified generic div remaining. The starting div-only version is:"
  solution={`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>City Library</title>
  </head>
  <body>
    <header>
      <h1>City Library</h1>
    </header>
    <nav>
      <a href="#hours">Hours</a> ·
      <a href="#events">Events</a>
    </nav>
    <main>
      <h2 id="hours">Opening hours</h2>
      <p>Open <div>Monday to Friday, 9 to 5</div></p>
    </main>
    <footer>
      <p>123 Main Street</p>
    </footer>
  </body>
</html>`}
/>

<Exercise
  title="A glossary with a description list"
  goal="Use a description list (dl) — a list type the core lessons skip — to pair each web term with its definition."
  steps={[
    "Create glossary.html with a valid skeleton and an h1 reading 'Web Glossary'.",
    "Add a dl element to hold the whole list.",
    "For each of the four terms, add a dt with the term, immediately followed by a dd with its definition.",
    "Cover these terms: HTML, element, attribute, tag.",
    "A dl is the right tool here because the data is genuinely term then definition — not plain bullets, which would be a ul.",
    "Validate, then commit with the message 'Add web glossary description list'."
  ]}
  expected="A page rendering four term/definition pairs, with each term slightly indented above its definition by default."
  solution={`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Web Glossary</title>
  </head>
  <body>
    <h1>Web Glossary</h1>
    <dl>
      <dt>HTML</dt>
      <dd>The language that describes the structure of a web page.</dd>
      <dt>Element</dt>
      <dd>A complete unit: an opening tag, content, and a closing tag.</dd>
      <dt>Attribute</dt>
      <dd>A name="value" pair inside an opening tag that configures the element.</dd>
      <dt>Tag</dt>
      <dd>The part written in angle brackets, such as opening and closing markers.</dd>
    </dl>
  </body>
</html>`}
/>

<Exercise
  title="A native FAQ with details and summary"
  goal="Build a collapsible FAQ using details and summary — interactive content with zero JavaScript."
  steps={[
    "Create faq.html with a valid skeleton and an h1 reading 'Plant Care FAQ'.",
    "For each question, add a details element.",
    "Inside each details, put the question in a summary element — that is the part that stays visible and toggles the panel.",
    "After the summary, add a p with the answer; it shows and hides when the summary is clicked.",
    "Add three questions in total, then open one with Live Server and confirm it expands and collapses on click.",
    "Validate, then commit with the message 'Add collapsible plant care FAQ'."
  ]}
  expected="Three rows that each expand to reveal an answer when clicked and collapse when clicked again — no JavaScript involved."
  solution={`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Plant Care FAQ</title>
  </head>
  <body>
    <h1>Plant Care FAQ</h1>
    <details>
      <summary>How often should I water a succulent?</summary>
      <p>Roughly every two weeks, once the soil is fully dry.</p>
    </details>
    <details>
      <summary>Why are the leaves turning yellow?</summary>
      <p>Usually overwatering. Let the soil dry out before the next watering.</p>
    </details>
    <details>
      <summary>How much light do they need?</summary>
      <p>Bright, indirect light for most of the day.</p>
    </details>
  </body>
</html>`}
/>

<Exercise
  title="A data table with scope and a spanning cell"
  goal="Extend a planet table with scope on its headers and one cell that spans multiple columns — both beyond the core table lesson."
  steps={[
    "Create planets.html with a valid skeleton and a table inside the body.",
    "Add a caption describing the table, then a thead with a header row.",
    "On each column header, add scope=\"col\" so assistive tech knows it labels the cells down its column.",
    "In each data row, make the planet name a th with scope=\"row\" so it labels the cells across its row.",
    "Add a final row whose single td uses colspan to span all the columns, holding a short summary line.",
    "Validate, then commit with the message 'Add planet table with scope and colspan'."
  ]}
  expected="A captioned table where every data cell is tied to both a row header and a column header, ending with one cell stretched across all columns."
  solution={`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Planet Data</title>
  </head>
  <body>
    <table>
      <caption>Inner planets at a glance</caption>
      <thead>
        <tr>
          <th scope="col">Planet</th>
          <th scope="col">Moons</th>
          <th scope="col">Day length (hours)</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <th scope="row">Mercury</th>
          <td>0</td>
          <td>4222</td>
        </tr>
        <tr>
          <th scope="row">Earth</th>
          <td>1</td>
          <td>24</td>
        </tr>
        <tr>
          <td colspan="3">Source: solar system fact sheets.</td>
        </tr>
      </tbody>
    </table>
  </body>
</html>`}
/>
