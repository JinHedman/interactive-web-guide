Every web page is an HTML document. Before a browser renders a single pixel, it reads a plain text file and builds a tree of elements. This chapter covers that foundational structure — the skeleton every page shares.

## The DOCTYPE declaration

The first line of every HTML file is a DOCTYPE declaration. It tells the browser which version of HTML you are using.

<CodeExample language="html">
{`<!doctype html>`}
</CodeExample>

For modern HTML (HTML5) the declaration is always exactly `<!doctype html>`. Nothing else is needed.

## The root element

After the DOCTYPE comes the `<html>` element. It wraps the entire document. The `lang` attribute tells browsers and assistive technologies which language the page is written in.

<CodeExample language="html">
{`<!doctype html>
<html lang="en">
</html>`}
</CodeExample>

Always include `lang`. It is a small addition with a large accessibility benefit.

## head and body

Inside `<html>` live exactly two children: `<head>` and `<body>`.

- `<head>` holds metadata — information *about* the page that the browser uses but does not display directly.
- `<body>` holds everything the user sees.

<CodeExample language="html">
{`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is my first web page.</p>
  </body>
</html>`}
</CodeExample>

Three things in `<head>` matter most:

1. `<meta charset="utf-8">` — ensures the browser uses the right character encoding (it handles accented letters, emoji, etc.).
2. `<meta name="viewport" ...>` — makes the page scale correctly on mobile screens.
3. `<title>` — the text that appears in the browser tab and in search results.

<ReadMore title="Where HTML goes next: components and templates">
Raw HTML files work, but modern web apps rarely write markup by hand for every page. Frameworks like **React** and **Vue** let you define reusable *components* — custom elements with their own logic and styles. React uses **JSX**, a syntax that looks like HTML but compiles to JavaScript. **Next.js** builds on React and adds routing, server rendering, and more. When you are comfortable with HTML structure, exploring JSX will feel familiar — the tags are the same; the mental model is what changes.
</ReadMore>
