HTML is the language that describes the structure of a web page: its headings, paragraphs, images, and links. Before a browser renders a single pixel, it reads your HTML file and builds a tree of elements from it. This chapter covers how to read any piece of HTML and how to write the skeleton every page shares. You will use it as the foundation for the profile page you build across this module.

## Elements, tags, and attributes

Here is one line of HTML. It looks small, but it has three named parts worth knowing.

<CodeExample language="html">
{`<a href="https://example.com">Visit my site</a>`}
</CodeExample>

- An **element** is the whole thing: the opening tag, the content, and the closing tag together. This is an anchor (link) element.
- A **tag** is the part in angle brackets. `<a>` is the opening tag; `</a>` is the closing tag (note the slash). The content goes between them.
- An **attribute** lives inside the opening tag and configures the element. Here `href="https://example.com"` sets the link's destination. Attributes are written as `name="value"`.

Most elements wrap content and need a closing tag. A few, like `<img>` and `<meta>`, are **empty elements** — they have no content and no closing tag, just attributes.

## Nesting

Elements go inside other elements, forming a tree. The rule is simple: tags must close in the reverse order they opened, like nested boxes.

<CodeExample language="html">
{`<p>My favorite language is <strong>JavaScript</strong>.</p>`}
</CodeExample>

The `<strong>` element is nested inside the `<p>` element. Indenting nested elements by two spaces keeps the structure readable as it grows.

## The DOCTYPE declaration

The first line of every HTML file is the DOCTYPE. It tells the browser to use modern (HTML5) rules.

<CodeExample language="html">
{`<!doctype html>`}
</CodeExample>

For modern HTML the declaration is always exactly `<!doctype html>`. Nothing else is needed, and it is not really a tag — it is a one-off instruction at the top of the file.

## The root element

After the DOCTYPE comes the `<html>` element, which wraps the entire document. The `lang` attribute states which language the page is written in.

<CodeExample language="html">
{`<!doctype html>
<html lang="en">
</html>`}
</CodeExample>

Always include `lang`. Screen readers use it to pick the right voice, and browsers use it for spell-check and translation. It is a tiny addition with a real accessibility payoff.

## head and body

Inside `<html>` live exactly two children: `<head>` and `<body>`. The split is about who the content is for.

- `<head>` holds **metadata** — information *about* the page that the browser uses but does not show on screen.
- `<body>` holds **content** — everything the visitor actually sees.

<CodeExample language="html" preview>
{`<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Ada's Profile</title>
  </head>
  <body>
    <h1>Ada Lovelace</h1>
    <p>I write algorithms and I am learning web development.</p>
  </body>
</html>`}
</CodeExample>

Three things in `<head>` matter most:

1. `<meta charset="utf-8">` sets the character encoding, so accented letters and emoji decode correctly.
2. `<meta name="viewport" ...>` makes the page scale properly on phones.
3. `<title>` is the text shown in the browser tab and in search-engine results. Notice it does not appear on the page itself — it is metadata, so it lives in `<head>`.

## Content vs metadata

A quick test for where something belongs: *will the visitor see it?* If yes, it goes in `<body>`. If it only describes or configures the page, it goes in `<head>`. The `<h1>` heading is visible, so it is content. The `<title>` shows in the tab, not the page, so it is metadata.
