<InShort>A `<form>` groups inputs; pair every `<input>` with a `<label>` via `for`/`id`, pick the right `type`, and add `<textarea>`, `<select>`, a `<button>`, and validation attributes like `required`.</InShort>

Forms are how a page collects input: a contact message, a sign-up, a search box. HTML gives you inputs for text, choices, and more, plus the labels and buttons that tie them together. In this chapter you add a contact form to your profile page. (Hooking the form up to actually do something comes later with JavaScript — here you build the markup.)

## The form element

A `<form>` wraps a group of inputs. Two attributes describe what happens when it is submitted:

- `action` — the URL the form data is sent to.
- `method` — how it is sent: `get` (appends data to the URL, for searches) or `post` (sends data in the request body, for anything that changes data).

<CodeExample language="html">
{`<form action="/contact" method="post">
  <!-- inputs go here -->
</form>`}
</CodeExample>

You will not have a real server to receive this yet, so submitting will not do much. The markup is what matters for now.

## Labels pair with inputs

<InShort>Give the input an `id`, point the label's `for` at that same id — now clicking the label focuses the input and screen readers announce it.</InShort>

Every input needs a `<label>`. Connect them by giving the input an `id` and pointing the label's `for` attribute at that same id. Clicking the label then focuses the input, and screen readers announce the label when the input is focused.

<CodeExample language="html" preview>
{`<form>
  <label for="name">Your name</label>
  <input type="text" id="name" name="name">
</form>`}
</CodeExample>

The `name` attribute is the key under which the value is submitted. The `for`/`id` pair is what makes the label and input one accessible unit. Always include both.

## Common input types

The `type` attribute changes how an `<input>` behaves and which on-screen keyboard mobile devices show.

<CodeExample language="html" preview>
{`<form>
  <label for="email">Email</label>
  <input type="email" id="email" name="email">

  <label for="pw">Password</label>
  <input type="password" id="pw" name="pw">

  <label for="age">Age</label>
  <input type="number" id="age" name="age">

  <p>Newsletter?
    <input type="checkbox" id="news" name="news">
    <label for="news">Yes, sign me up</label>
  </p>

  <p>Experience level:</p>
  <input type="radio" id="beginner" name="level" value="beginner">
  <label for="beginner">Beginner</label>
  <input type="radio" id="pro" name="level" value="pro">
  <label for="pro">Pro</label>
</form>`}
</CodeExample>

A note on radio buttons: give every option in a group the **same `name`** so only one can be selected at a time. Each option's `value` is what gets submitted.

## Textareas and selects

For multi-line text, use `<textarea>` (which has a closing tag, unlike `<input>`). For a dropdown of choices, use `<select>` with an `<option>` for each choice.

<CodeExample language="html" preview>
{`<form>
  <label for="message">Message</label>
  <textarea id="message" name="message" rows="4"></textarea>

  <label for="topic">Topic</label>
  <select id="topic" name="topic">
    <option value="job">Job opportunity</option>
    <option value="collab">Collaboration</option>
    <option value="hello">Just saying hi</option>
  </select>
</form>`}
</CodeExample>

Each `<option>` carries a `value` that is submitted; the text between the tags is what the visitor sees.

## Buttons and basic validation

A `<button>` submits or resets the form, or runs JavaScript. The `type` attribute decides which:

- `type="submit"` (the default inside a form) sends the form.
- `type="button"` does nothing on its own — you wire it up with JavaScript later.
- `type="reset"` clears the fields.

You can add lightweight validation right in the markup. `required` blocks submission if the field is empty, and `placeholder` shows greyed-out hint text inside the field.

<CodeExample language="html" preview>
{`<form>
  <label for="contact-email">Email</label>
  <input type="email" id="contact-email" name="email"
         placeholder="you@example.com" required>

  <button type="submit">Send</button>
</form>`}
</CodeExample>

`placeholder` is a hint, not a label — it disappears as soon as the visitor types, so it can never replace a real `<label>`.
