Text tells people who you are; an image lets them see it in a glance. A photo, a logo, a screenshot of your work — media is what turns a wall of words into a page people actually want to look at. HTML embeds images, audio, and video with a small set of elements, and the key skill is doing it accessibly: describing images so everyone, including people using a screen reader, can understand them. In this chapter you add a profile photo and learn the media basics.

## The img element

An image is an `<img>` element. It is an empty element — no closing tag — configured entirely by attributes. The two essential ones are `src` (where the image is) and `alt` (a text description).

<CodeExample language="html" preview>
{`<img
  src="https://placehold.co/200x200"
  alt="Portrait of Ada Lovelace"
  width="200"
  height="200">`}
</CodeExample>

`src` is the image's address. `alt` is alternative text, read aloud by screen readers and shown if the image fails to load.

## Relative paths to local images

For images stored in your own project, use a relative path, just like relative links. Keep images in a folder such as `images/` to stay organized.

<CodeExample language="html">
{`<!-- image file lives at images/profile.jpg next to index.html -->
<img src="images/profile.jpg" alt="A photo of me at the beach">`}
</CodeExample>

The path is relative to the HTML file. `images/profile.jpg` means "the `images` folder beside this page, then `profile.jpg` inside it."

## Good alt text, and when it should be empty

Write `alt` as a short description of what the image shows and why it matters here — not the word "image" (the browser already knows it is one).

There is one exception. If an image is purely **decorative** and adds no information — a divider flourish, a background swirl — give it an empty `alt=""`. That tells screen readers to skip it rather than announce a meaningless filename.

<CodeExample language="html">
{`<!-- Informative: describe it -->
<img src="images/profile.jpg" alt="Me speaking at a meetup">

<!-- Decorative: empty alt so screen readers skip it -->
<img src="images/divider.png" alt="">`}
</CodeExample>

Every `<img>` needs an `alt` attribute. The choice is between a real description and an intentionally empty one — never leave it off.

## width and height reduce layout shift

Set `width` and `height` attributes (in pixels, no units) on images. They let the browser reserve the right amount of space before the image downloads, so text does not jump around as images load. That jumping is called layout shift, and it is annoying.

<CodeExample language="html">
{`<img src="images/profile.jpg" alt="A photo of me" width="320" height="240">`}
</CodeExample>

These set the image's intrinsic size. You can still resize it later with CSS; the attributes mainly tell the browser the aspect ratio up front.

## Audio and video

Embed sound with `<audio>` and video with `<video>`. Add the `controls` attribute so the visitor gets play, pause, and volume controls.

<CodeExample language="html">
{`<audio src="audio/intro.mp3" controls></audio>

<video src="video/demo.mp4" controls width="480"></video>`}
</CodeExample>

Both elements need a closing tag. Without `controls` the player shows no buttons, so include it unless you plan to control playback with JavaScript later.
