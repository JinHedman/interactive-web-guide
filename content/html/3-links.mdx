<InShort>An `<a>` element with an `href` is a link; `href` can be **absolute** (another site), **relative** (your own pages), or a `#fragment` (a section of the same page).</InShort>

Links are what make the web a web — without them, every page would be an island. The anchor element, `<a>`, connects your page to other pages, to external sites, and even to specific spots within the same page, which is how a navigation menu jumps a reader straight to a section. In this chapter you add links and in-page navigation to your profile page.

## The anchor element

A link is an `<a>` element with an `href` attribute. `href` ("hypertext reference") is the destination; the content between the tags is the clickable text.

<CodeExample language="html" preview>
{`<p>Find me on <a href="https://github.com">GitHub</a>.</p>`}
</CodeExample>

The text inside the tags is what the visitor clicks. The `href` is where they go.

## Relative vs absolute URLs

The value of `href` can point two ways:

- An **absolute URL** includes the full address with the scheme, like `https://example.com/about`. Use it to link to other sites.
- A **relative URL** is a path relative to the current page, like `about.html` or `projects/web.html`. Use it to link within your own site.

<CodeExample language="html">
{`<!-- Absolute: another website -->
<a href="https://developer.mozilla.org">MDN Web Docs</a>

<!-- Relative: another page in your own project -->
<a href="projects.html">My projects</a>`}
</CodeExample>

Relative links keep working if you move the whole site to a new domain, because they do not hard-code the host. Reach for relative links inside your own project and absolute links for the outside world.

## Linking within a page

To jump to a section of the *same* page, give the target element an `id`, then link to it with `#` plus that id. This is called a fragment link.

<CodeExample language="html" preview>
{`<nav>
  <a href="#about">About</a> ·
  <a href="#projects">Projects</a>
</nav>

<h2 id="about">About me</h2>
<p>A short bio goes here.</p>

<h2 id="projects">Projects</h2>
<p>A list of things I've built.</p>`}
</CodeExample>

An `id` must be unique on the page. Clicking the link scrolls to the element whose `id` matches the fragment. This is exactly how a "jump to section" menu works.

## Opening in a new tab

To open a link in a new browser tab, add `target="_blank"`. When you do, also add `rel="noopener"`.

<CodeExample language="html">
{`<a href="https://example.com" target="_blank" rel="noopener">My portfolio</a>`}
</CodeExample>

Why `rel="noopener"`? Without it, the page you open can gain a reference back to your page through `window.opener`, which is a security and performance risk. `rel="noopener"` severs that link. Make it a habit: `target="_blank"` and `rel="noopener"` travel together.

## Write link text that stands alone

Link text should describe its destination on its own, without surrounding sentences. Screen-reader users often jump from link to link, so a list of "click here" links tells them nothing.

<CodeExample language="html">
{`<!-- Weak: meaningless out of context -->
<p>To see my work, <a href="projects.html">click here</a>.</p>

<!-- Strong: the link text describes the destination -->
<p>Browse <a href="projects.html">my recent projects</a>.</p>`}
</CodeExample>

Aim for link text that names where it goes. It is better for accessibility and reads more naturally too.
