When you need to do the same work many times — print every hobby in a list, count from 1 to 10, total up some numbers — you use a *loop*. This chapter covers the four loops you will use most, and the one mistake that locks up the page.

## The for loop

A classic `for` loop has three parts in its parentheses: a starting counter, a condition to keep going, and a step that runs after each pass.

<CodeExample language="js" preview>
{`for (let i = 0; i < 5; i++) {
  console.log(\`Pass number \${i}\`);
}
// Pass number 0
// Pass number 1
// ... up to 4`}
</CodeExample>

Read it as: start `i` at `0`; keep looping **while** `i < 5`; after each pass, do `i++` (add one to `i`). When the condition becomes false, the loop stops. The counter is handy when you need the index — for example, to read array items by position.

<CodeExample language="js" preview>
{`const hobbies = ["reading", "cycling", "cooking"];

for (let i = 0; i < hobbies.length; i++) {
  console.log(\`\${i + 1}. \${hobbies[i]}\`);
}
// 1. reading
// 2. cycling
// 3. cooking`}
</CodeExample>

Looping while `i < hobbies.length` works for any array length — you do not hard-code the count.

## for...of

When you just want each item and do not care about the index, `for...of` is cleaner.

<CodeExample language="js" preview>
{`const hobbies = ["reading", "cycling", "cooking"];

for (const hobby of hobbies) {
  console.log(hobby);
}
// reading
// cycling
// cooking`}
</CodeExample>

Each pass puts the next item into `hobby`. No counter, no `[i]` — less to get wrong. Reach for `for...of` whenever you are walking through an array's values.

## The while loop

A `while` loop repeats as long as its condition stays true. Use it when you do not know the number of passes up front.

<CodeExample language="js" preview>
{`let countdown = 3;

while (countdown > 0) {
  console.log(countdown);
  countdown--; // this is what eventually ends the loop
}
console.log("Lift off!");
// 3
// 2
// 1
// Lift off!`}
</CodeExample>

**Watch out for infinite loops.** Something inside the loop must eventually make the condition false. If you forget `countdown--`, the condition stays `true` forever and the page freezes. Always make sure each pass moves toward the exit.

## forEach

Arrays have a `forEach` method that runs a function once per item. You will see this style constantly in real code.

<CodeExample language="js" preview>
{`const hobbies = ["reading", "cycling", "cooking"];

hobbies.forEach(function (hobby, index) {
  console.log(\`\${index}: \${hobby}\`);
});
// 0: reading
// 1: cycling
// 2: cooking`}
</CodeExample>

`forEach` hands your function each item (and optionally its index). It is concise for "do this with every element." The function you pass is covered properly in the next chapter — here, just notice the pattern.
