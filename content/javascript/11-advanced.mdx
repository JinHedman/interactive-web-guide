<InShort>Optional and deeper: transform data with `map`, `filter`, and `reduce`, and write cleaner code with destructuring, spread, and optional chaining — the everyday toolkit of modern JavaScript and React.</InShort>

This is an **optional, deeper chapter**. Everything in chapters 1–10 stands on its own — you can already loop, branch, and write functions. But modern JavaScript leans hard on a smaller, sharper toolkit for handling data: a few array methods, some syntax shortcuts, and safe ways to read data that might be missing. These are the patterns you will see in almost every real codebase and every React tutorial. None of it is *new* power — you could do all of it with the loops you already know — but it is shorter, clearer, and the common language of the field.

Each section teaches one tool with a runnable example, then you will drill them and finish with a quiz. Take it at your own pace.

<WhyItMatters>
You can write all of this with the loops you already know — so why learn it? Because it is the language real teams read and write every day, and React in particular is built around it (you render a list by `map`-ing data straight to UI). Picking these up in plain JavaScript now means a framework later feels like a small step, not a new language.
</WhyItMatters>

## map: transform every item

<InShort>`map` takes an array and returns a **new** array of the same length, with each item transformed by the callback you pass — the original is untouched.</InShort>

You already know how to loop over an array and do something with each item (chapter 6, `for...of` and `forEach`). `map` is the same idea with a sharper purpose: **take an array, return a new array of the same length, with each item transformed.**

It takes a function — a *callback* (chapter 7) — that runs once per item. Whatever that function returns becomes the matching item in the new array.

<CodeExample language="js" preview>
{`const prices = [4, 9, 2, 6];

// The old way you already know:
const withTaxLoop = [];
for (const p of prices) {
  withTaxLoop.push(p * 1.25);
}

// The same thing with map — shorter and clearer:
const withTax = prices.map((p) => p * 1.25);

console.log(withTax);     // [5, 11.25, 2.5, 7.5]
console.log(prices);      // [4, 9, 2, 6] — unchanged`}
</CodeExample>

Two things to notice. First, `map` does not change the original array — it *returns a new one*, so `prices` is untouched. Second, the arrow callback `(p) => p * 1.25` runs for each item, and its return value fills the new array. `map` is the declarative version of the "build a new array in a loop" pattern.

It shines on arrays of objects — pull one field out of each:

<CodeExample language="js" preview>
{`const people = [
  { name: "Ada", age: 30 },
  { name: "Grace", age: 45 },
  { name: "Linus", age: 28 }
];

const names = people.map((person) => person.name);
console.log(names); // ["Ada", "Grace", "Linus"]`}
</CodeExample>

## filter: keep only some items

<InShort>`filter` returns a new array of just the items for which your callback returns `true` — the callback is a yes/no **test** run on each item.</InShort>

Where `map` returns one item per input, `filter` returns *fewer* items: it keeps only the ones for which your callback returns `true`. The callback is a **test**.

<CodeExample language="js" preview>
{`const prices = [4, 9, 2, 6];

// Keep only the prices over 5:
const expensive = prices.filter((p) => p > 5);
console.log(expensive); // [9, 6]

const people = [
  { name: "Ada", age: 30 },
  { name: "Grace", age: 45 },
  { name: "Linus", age: 28 }
];

// Keep only people aged 30 or over:
const thirtyPlus = people.filter((person) => person.age >= 30);
console.log(thirtyPlus.length); // 2`}
</CodeExample>

Read the callback as a yes/no question for each item: "is this price over 5?" Items that answer `true` make it into the new array; the rest are dropped. Like `map`, `filter` never changes the original.

`map` and `filter` *chain* — each returns an array, so you can call the next method right on the result:

<CodeExample language="js" preview>
{`const prices = [4, 9, 2, 6];

// Keep the prices over 5, then add tax to those:
const result = prices
  .filter((p) => p > 5)
  .map((p) => p * 1.25);

console.log(result); // [11.25, 7.5]`}
</CodeExample>

## reduce: collapse to a single value

<InShort>`reduce` collapses a whole array into **one value** — a total, a max, a joined string — by carrying a running *accumulator* from item to item.</InShort>

`map` and `filter` give you back an array. `reduce` is for when you want **one value** out of a whole array — a total, a maximum, a joined-up string.

`reduce` is the trickiest of the three, so go slow. Its callback takes *two* arguments: the **accumulator** (the running result so far) and the **current item**. You also pass a starting value as the second argument to `reduce`. Whatever the callback returns becomes the accumulator for the next item.

<CodeExample language="js" preview>
{`const prices = [4, 9, 2, 6];

// Sum them. Start the running total at 0.
const total = prices.reduce((runningTotal, price) => {
  return runningTotal + price;
}, 0);

console.log(total); // 21`}
</CodeExample>

Walk through it: the accumulator starts at `0`. Then it becomes `0 + 4 = 4`, then `4 + 9 = 13`, then `13 + 2 = 15`, then `15 + 6 = 21`. The final accumulator is the answer.

The same loop written the long way makes the pattern obvious — `reduce` is exactly "a running variable updated in a loop," packaged up:

<CodeExample language="js" preview>
{`const prices = [4, 9, 2, 6];

// What reduce is doing under the hood:
let total = 0;
for (const price of prices) {
  total = total + price;
}
console.log(total); // 21`}
</CodeExample>

`reduce` works on objects too. Total a shopping cart by reaching into each item:

<CodeExample language="js" preview>
{`const cart = [
  { item: "book", price: 12 },
  { item: "pen", price: 3 },
  { item: "mug", price: 8 }
];

const grandTotal = cart.reduce((sum, product) => sum + product.price, 0);
console.log(grandTotal); // 23`}
</CodeExample>

## find, sort, and join

Three more everyday methods worth knowing.

`find` returns the **first** item that passes a test — the whole item, not a new array. It is what you reach for to look something up by name or id.

<CodeExample language="js" preview>
{`const people = [
  { name: "Ada", age: 30 },
  { name: "Grace", age: 45 }
];

const grace = people.find((person) => person.name === "Grace");
console.log(grace.age); // 45

const missing = people.find((person) => person.name === "Nobody");
console.log(missing); // undefined — find returns undefined if nothing matches`}
</CodeExample>

`sort` reorders an array. For numbers, pass a compare function: return a negative number to put `a` first, positive to put `b` first. `(a, b) => a - b` sorts ascending. Note that `sort` **changes the array in place**, so copy first (with spread, below) if you need the original order.

<CodeExample language="js" preview>
{`const scores = [42, 8, 100, 16];

const ascending = [...scores].sort((a, b) => a - b);
console.log(ascending); // [8, 16, 42, 100]

const descending = [...scores].sort((a, b) => b - a);
console.log(descending); // [100, 42, 16, 8]`}
</CodeExample>

`join` turns an array into a single string, with a separator you choose.

<CodeExample language="js" preview>
{`const names = ["Ada", "Grace", "Linus"];

console.log(names.join(", ")); // "Ada, Grace, Linus"
console.log(names.join(" — ")); // "Ada — Grace — Linus"`}
</CodeExample>

## Destructuring: pull values out by name

<InShort>**Destructuring** pulls fields out of an object (by key, with `{ }`) or an array (by position, with `[ ]`) into their own variables in a single line.</InShort>

You often want to copy a few fields out of an object into their own variables. The long way repeats the object name each time; **destructuring** does it in one line.

<CodeExample language="js" preview>
{`const person = { name: "Ada", age: 30, city: "London" };

// The long way:
const name1 = person.name;
const age1 = person.age;

// Destructuring — pull name and age out by their keys:
const { name, age } = person;
console.log(name); // Ada
console.log(age);  // 30`}
</CodeExample>

The names in the braces must match the object's keys. Arrays destructure too, but **by position**, using square brackets:

<CodeExample language="js" preview>
{`const colors = ["red", "green", "blue"];

const [first, second] = colors;
console.log(first);  // red
console.log(second); // green`}
</CodeExample>

Destructuring is especially tidy inside a callback — pull the fields you need straight from the parameter:

<CodeExample language="js" preview>
{`const people = [
  { name: "Ada", age: 30 },
  { name: "Grace", age: 45 }
];

// Destructure each person right in the map callback:
const lines = people.map(({ name, age }) => \`\${name} is \${age}\`);
console.log(lines); // ["Ada is 30", "Grace is 45"]`}
</CodeExample>

## Spread: copy and combine

<InShort>The spread operator `...` copies the items of an array or the keys of an object into a new one — the clean way to copy or combine collections without changing the original.</InShort>

The spread operator `...` "spreads out" the items of an array or the keys of an object into a new one. It is the clean way to **copy** a collection or build a new one with extra items, without changing the original.

<CodeExample language="js" preview>
{`const tasks = ["write", "review"];

// Build a NEW array with one more item — tasks is untouched:
const more = [...tasks, "ship"];
console.log(more);  // ["write", "review", "ship"]
console.log(tasks); // ["write", "review"] — unchanged

// Combine two arrays:
const a = [1, 2];
const b = [3, 4];
console.log([...a, ...b]); // [1, 2, 3, 4]`}
</CodeExample>

Objects spread the same way. This is how you make a copy with one field changed — the pattern at the heart of updating state in React:

<CodeExample language="js" preview>
{`const person = { name: "Ada", age: 30 };

// Copy person, but override age. The original is untouched.
const older = { ...person, age: 31 };

console.log(older);  // { name: "Ada", age: 31 }
console.log(person); // { name: "Ada", age: 30 } — unchanged`}
</CodeExample>

Keys later in the literal win, so `{ ...person, age: 31 }` copies everything from `person` and then overrides `age`.

## Optional chaining and nullish coalescing

<InShort>Optional chaining `?.` reads a property only if what precedes it exists (otherwise `undefined`, no crash); nullish coalescing `??` supplies a fallback only when a value is `null` or `undefined`.</InShort>

Real data is often incomplete — a person might have no address, an address might have no postcode. Reading `person.address.city` when `address` is missing throws the classic `Cannot read properties of undefined` error you met in the DevTools chapter.

**Optional chaining** `?.` reads a property only if the thing before it exists. If any link in the chain is `null` or `undefined`, the whole expression stops and gives back `undefined` instead of crashing.

<CodeExample language="js" preview>
{`const ada = { name: "Ada", address: { city: "London" } };
const grace = { name: "Grace" }; // no address

console.log(ada.address?.city);   // "London"
console.log(grace.address?.city); // undefined — no crash`}
</CodeExample>

Without the `?.`, `grace.address.city` would throw, because `grace.address` is `undefined` and you cannot read `.city` from `undefined`.

**Nullish coalescing** `??` supplies a fallback when a value is `null` or `undefined`. Pair it with `?.` to read safely *and* default in one expression:

<CodeExample language="js" preview>
{`const grace = { name: "Grace" }; // no address

const city = grace.address?.city ?? "Unknown";
console.log(city); // "Unknown"`}
</CodeExample>

Why `??` and not `||`? `||` falls back on *any* falsy value, including `0` and `""` (empty string), which are often valid data. `??` falls back only on `null` or `undefined`, so a real `0` survives:

<CodeExample language="js" preview>
{`const count = 0;

console.log(count || 5); // 5  — || treats 0 as "missing" (often wrong)
console.log(count ?? 5); // 0  — ?? keeps a real 0`}
</CodeExample>

## Iterating objects: Object.keys, values, and entries

Arrays loop naturally, but objects do not — there is no index. To walk an object's properties, turn it into an array first with one of three helpers, then use `map`, `filter`, or `for...of`.

<CodeExample language="js" preview>
{`const skills = { html: 5, css: 3, js: 4 };

console.log(Object.keys(skills));   // ["html", "css", "js"]
console.log(Object.values(skills)); // [5, 3, 4]
console.log(Object.entries(skills)); // [["html", 5], ["css", 3], ["js", 4]]`}
</CodeExample>

`Object.entries` is the most useful: it gives an array of `[key, value]` pairs, which you can destructure in a loop to handle both at once.

<CodeExample language="js" preview>
{`const skills = { html: 5, css: 3, js: 4 };

// Destructure each [key, value] pair:
for (const [skill, level] of Object.entries(skills)) {
  console.log(\`\${skill}: \${level}\`);
}
// html: 5
// css: 3
// js: 4

// Or sum the values with map + reduce:
const total = Object.values(skills).reduce((sum, n) => sum + n, 0);
console.log(total); // 12`}
</CodeExample>

<ReadMore title="Where this goes next: React and the declarative mindset">
The instinct you just practiced — *describe a transformation of your data instead of mutating things step by step* — is the whole idea behind **React**. In React you render a list by mapping an array straight to UI: `items.map((item) => <li key={item.id}>{item.text}</li>)`. The `map` is exactly the one you learned here; the destructuring (`{ name, age }`) is how you read a component's *props*; and the spread copy (`{ ...state, done: true }`) is how you update state without mutating it, because React compares the new object to the old one to decide what to redraw. Optional chaining and `??` show up constantly when rendering data fetched from a server that might not have arrived yet. Learn these in plain JavaScript first — they are not React features, just modern JavaScript — and React will feel like a natural next step rather than a new language.

- [MDN – Array.prototype.map()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map) — the canonical reference, with more examples.
- [React – Rendering Lists](https://react.dev/learn/rendering-lists) — see `map` used to build UI from an array.
- [javascript.info – Destructuring assignment](https://javascript.info/destructuring-assignment) — a thorough walkthrough of destructuring and spread.
</ReadMore>
