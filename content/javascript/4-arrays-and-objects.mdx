A single variable holds one value. Real programs work with *collections*: a list of hobbies, a person's details, a set of to-do items. JavaScript has two core structures for this — **arrays** for ordered lists and **objects** for labelled values.

## Arrays: ordered lists

An array is an ordered list, written with square brackets. You read items by their *index* — and indexes start at `0`.

<CodeExample language="js" preview>
{`const hobbies = ["reading", "cycling", "cooking"];

console.log(hobbies[0]); // reading  — the first item
console.log(hobbies[2]); // cooking  — the third item
console.log(hobbies.length); // 3     — how many items`}
</CodeExample>

The first item is at index `0`, so the last item of a list with `length` 3 is at index `2`. `length` always tells you how many items the array holds.

## Adding and removing items

`push` adds to the end of an array; `pop` removes the last item and hands it back.

<CodeExample language="js" preview>
{`const hobbies = ["reading", "cycling"];

hobbies.push("cooking");
console.log(hobbies);        // ["reading", "cycling", "cooking"]
console.log(hobbies.length); // 3

const removed = hobbies.pop();
console.log(removed);        // cooking
console.log(hobbies);        // ["reading", "cycling"]`}
</CodeExample>

Notice these change the array in place even though it is declared with `const`. `const` stops you from *reassigning* the variable to a whole new array — it does not freeze the contents.

One note on the Result pane: when you log an array or object, the preview shows it as formatted JSON — keys in double quotes, spread over multiple lines. That is just how the preview displays it; the inline comments here use the shorter JavaScript-literal form you would type in code.

## Objects: labelled values

An object groups related values under named *keys*, written with curly braces. Each entry is a `key: value` pair.

<CodeExample language="js" preview>
{`const person = {
  name: "Ada",
  age: 30,
  isMember: true
};

console.log(person.name);  // Ada
console.log(person.age);   // 30`}
</CodeExample>

Where an array answers "what is at position 2?", an object answers "what is the `name`?". Use an array for a list of similar things, and an object for one thing with several named attributes.

## Dot vs bracket notation

There are two ways to read or set a property.

<CodeExample language="js" preview>
{`const person = { name: "Ada", age: 30 };

// Dot notation — the usual choice
console.log(person.name);   // Ada

// Bracket notation — needed when the key is in a variable
const key = "age";
console.log(person[key]);   // 30

// Updating and adding properties
person.age = 31;            // update
person.city = "London";     // add a new key
console.log(person);        // { name: "Ada", age: 31, city: "London" }`}
</CodeExample>

Use dot notation by default. Reach for bracket notation when the key comes from a variable, or when a key contains spaces or other awkward characters.

## Nesting

Arrays and objects can hold each other, which is how you model anything realistic — a list of people, where each person has a list of hobbies.

<CodeExample language="js" preview>
{`const person = {
  name: "Ada",
  hobbies: ["reading", "cycling"],
  address: {
    city: "London",
    country: "UK"
  }
};

console.log(person.hobbies[0]);     // reading
console.log(person.address.city);   // London
console.log(person.hobbies.length); // 2`}
</CodeExample>

To reach deep into a structure, chain the steps: `person.address.city` reads the `address` object, then its `city` key. `person.hobbies[0]` reads the `hobbies` array, then its first item.
