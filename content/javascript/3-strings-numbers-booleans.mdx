Now that you can store values, this chapter is about working with them: joining and inspecting text, doing reliable number work, converting between the two, and understanding why `if` statements treat some values as "true" even when they are not literally `true`.

## Joining strings

The simplest way to combine strings is with `+`, called *concatenation*.

<CodeExample language="js" preview>
{`const first = "Ada";
const last = "Lovelace";

const fullName = first + " " + last;
console.log(fullName); // Ada Lovelace`}
</CodeExample>

That works, but the spaces and `+` signs get fiddly. **Template literals** are cleaner. Use backticks (`` ` ``) and drop variables inside `${...}`.

<CodeExample language="js" preview>
{`const name = "Ada";
const count = 3;

const message = \`\${name} has \${count} hobbies.\`;
console.log(message); // Ada has 3 hobbies.`}
</CodeExample>

Template literals can also span multiple lines, which plain strings cannot. Prefer them whenever you mix text and variables.

## Useful string methods

A string carries built-in tools. You call them with a dot after the value.

<CodeExample language="js" preview>
{`const hobby = "Photography";

console.log(hobby.length);          // 11  — number of characters
console.log(hobby.toUpperCase());   // PHOTOGRAPHY
console.log(hobby.toLowerCase());   // photography
console.log(hobby.includes("graph")); // true — does it contain this?
console.log(hobby.slice(0, 5));     // Photo — characters 0 up to (not including) 5`}
</CodeExample>

- `length` is a property (no parentheses) — the character count.
- `toUpperCase()` / `toLowerCase()` return a new string in that case.
- `includes("...")` returns a boolean: does the string contain that text?
- `slice(start, end)` returns the piece from `start` up to (but not including) `end`.

These methods **return a new value** — they do not change the original string.

## Number work and conversion

Numbers use the arithmetic from the last chapter. The tricky part is that input from a page always arrives as a *string*, so `"5"` and `5` are different. You convert with `Number(...)` or `parseInt(...)`.

<CodeExample language="js" preview>
{`const typed = "42";          // a string, e.g. from a form
console.log(typed + 1);      // "421" — string concatenation, not math!

const asNumber = Number(typed);
console.log(asNumber + 1);   // 43 — now it's real addition

console.log(parseInt("12px"));   // 12 — parseInt stops at the first non-digit
console.log(Number("12px"));     // NaN — Number is stricter`}
</CodeExample>

- `Number("42")` converts a clean numeric string to a number; anything non-numeric becomes `NaN` ("not a number").
- `parseInt("12px")` reads digits from the front and ignores the rest — useful for values like `"12px"`.

When you accidentally add a number to a string, JavaScript turns the number into a string and joins them. Converting first avoids that bug.

## Truthy and falsy

In a condition, JavaScript treats every value as either *truthy* or *falsy*. Only a short list of values are **falsy**:

<CodeExample language="js" preview>
{`// These six values are falsy:
console.log(Boolean(false));     // false
console.log(Boolean(0));         // false
console.log(Boolean(""));        // false (empty string)
console.log(Boolean(null));      // false
console.log(Boolean(undefined)); // false
console.log(Boolean(NaN));       // false

// Almost everything else is truthy:
console.log(Boolean("hello"));   // true
console.log(Boolean(42));        // true`}
</CodeExample>

So `0`, `""`, `null`, `undefined`, `NaN`, and `false` are falsy; everything else — any non-empty string, any non-zero number — is truthy. You will lean on this in the next chapter when writing conditions like `if (name)` to mean "if name has a value."
