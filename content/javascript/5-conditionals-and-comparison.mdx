So far your code runs straight through, top to bottom. *Conditionals* let it choose: run this block if a condition holds, otherwise run another. The condition is usually a comparison between two values, and this chapter covers both the comparisons and the branching.

## Comparing values

Comparison operators return a boolean — `true` or `false`.

<CodeExample language="js" preview>
{`console.log(5 > 3);    // true
console.log(5 < 3);    // false
console.log(5 >= 5);   // true  (greater than or equal)
console.log(4 <= 3);   // false (less than or equal)
console.log(5 === 5);  // true  (strictly equal)
console.log(5 !== 3);  // true  (not equal)`}
</CodeExample>

Use `===` to test equality and `!==` to test inequality. These are the **strict** operators: they compare both value and type, with no surprise conversions.

There is also a loose `==`. **Avoid it.** It converts types before comparing, which leads to confusing results like `0 == ""` being `true`. Always use `===` and `!==`.

<CodeExample language="js" preview>
{`console.log(1 === "1");  // false — different types (number vs string)
console.log(1 == "1");   // true  — == converts the string first (avoid this!)`}
</CodeExample>

## Combining conditions

Three logical operators let you build bigger conditions:

- `&&` (AND) — true only if **both** sides are true.
- `||` (OR) — true if **either** side is true.
- `!` (NOT) — flips a boolean.

<CodeExample language="js" preview>
{`const age = 20;
const hasTicket = true;

console.log(age >= 18 && hasTicket); // true  — both conditions hold
console.log(age < 13 || hasTicket);  // true  — at least one holds
console.log(!hasTicket);             // false — flips true to false`}
</CodeExample>

## if / else if / else

`if` runs a block when its condition is truthy. Add `else if` for more cases and `else` as a fallback.

<CodeExample language="js" preview>
{`const score = 72;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 70) {
  console.log("Grade: B");
} else if (score >= 50) {
  console.log("Grade: C");
} else {
  console.log("Grade: F");
}
// Grade: B`}
</CodeExample>

JavaScript checks each condition top to bottom and runs the **first** block that matches, then skips the rest. Order matters: put the most specific or highest-priority condition first.

Remember truthiness from the strings chapter — you can test a value directly:

<CodeExample language="js" preview>
{`const name = "";

if (name) {
  console.log(\`Hello, \${name}\`);
} else {
  console.log("No name provided"); // runs, because "" is falsy
}`}
</CodeExample>

## The ternary operator

For a simple either/or choice, a *ternary* is a compact one-line alternative to `if`/`else`. Its shape is `condition ? valueIfTrue : valueIfFalse`.

<CodeExample language="js" preview>
{`const age = 20;

const status = age >= 18 ? "adult" : "minor";
console.log(status); // adult

const count = 1;
const label = count === 1 ? "item" : "items";
console.log(\`\${count} \${label}\`); // 1 item`}
</CodeExample>

Use a ternary when you are choosing between two values. For anything more involved, a full `if`/`else if`/`else` stays more readable.
