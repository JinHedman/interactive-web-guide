A variable is a name for a value. Instead of repeating `"Ada"` everywhere, you store it once and refer to it by name. This chapter covers how to declare variables, the basic kinds of values JavaScript holds, and the operators that work on them.

## Declaring variables

You create a variable with `const` or `let`, followed by a name, an `=`, and a value.

<CodeExample language="js" preview>
{`const name = "Ada";
let age = 30;

console.log(name);
console.log(age);`}
</CodeExample>

- `const` creates a value you will **not reassign**. Try to change it and JavaScript throws an error.
- `let` creates a value you **can reassign** later.

<CodeExample language="js" preview>
{`let score = 10;
console.log(score);

score = 25; // reassigning is fine with let
console.log(score);`}
</CodeExample>

**Rule of thumb: reach for `const` by default.** Use `let` only when you know the value will change. This makes your intent obvious and prevents accidental changes.

You may see older code use `var`. Avoid it — its scoping rules are surprising and `let`/`const` replaced it. If you write new code, you do not need `var`.

## The primitive types

A *type* describes what kind of value you have. JavaScript has a handful of basic, or *primitive*, types. These five cover almost everything you will do at first:

<CodeExample language="js" preview>
{`const title = "Hobbies";     // string  — text, in quotes
const count = 3;              // number  — integer or decimal
const isPublic = true;        // boolean — true or false
const middleName = null;      // null    — intentionally empty
let nickname;                 // undefined — declared but no value yet

console.log(title, count, isPublic, middleName, nickname);`}
</CodeExample>

- **string**: text wrapped in quotes — `"hello"` or `'hello'`.
- **number**: any number, whole or decimal — `42`, `3.14`. JavaScript has just one number type.
- **boolean**: `true` or `false`.
- **null**: a deliberate "no value." You set it on purpose.
- **undefined**: a variable that exists but was never given a value.

You can check a value's type with `typeof`.

<CodeExample language="js" preview>
{`console.log(typeof "hello");  // "string"
console.log(typeof 42);       // "number"
console.log(typeof true);     // "boolean"
console.log(typeof undefined);// "undefined"`}
</CodeExample>

## Arithmetic operators

Numbers support the math operators you would expect.

<CodeExample language="js" preview>
{`const a = 7;
const b = 2;

console.log(a + b);  // 9   addition
console.log(a - b);  // 5   subtraction
console.log(a * b);  // 14  multiplication
console.log(a / b);  // 3.5 division
console.log(a % b);  // 1   remainder (modulo)`}
</CodeExample>

The `%` (modulo) operator gives the remainder after division. It is handy for "every Nth time" logic and checking if a number is even (`n % 2 === 0`).

## Assignment operators

The plain `=` assigns a value. There are shorthands for updating a variable based on its current value.

<CodeExample language="js" preview>
{`let total = 100;

total += 20; // same as: total = total + 20
console.log(total); // 120

total -= 50; // total = total - 50
console.log(total); // 70

total *= 2;  // total = total * 2
console.log(total); // 140`}
</CodeExample>

These only work with `let`, since they reassign the variable.

## Increment and decrement

Adding or subtracting one is so common it has its own shorthand: `++` increases a variable by one, and `--` decreases it by one. You will see `i++` constantly in loops, where it steps a counter forward each pass.

<CodeExample language="js" preview>
{`let i = 0;

i++; // same as i += 1
console.log(i); // 1

i++;
console.log(i); // 2

i--; // same as i -= 1
console.log(i); // 1`}
</CodeExample>

Like the other assignment operators, `++` and `--` only work with `let`.
