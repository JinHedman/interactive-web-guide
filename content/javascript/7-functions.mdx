A function is a named block of code you can run whenever you want, as many times as you want. Functions are how you avoid repeating yourself and how you break a program into understandable pieces. This is the most important chapter in the module — almost everything you write from here on lives inside functions.

## Declaring and calling

You *declare* a function with the `function` keyword, then *call* it by writing its name followed by parentheses.

<CodeExample language="js" preview>
{`function greet() {
  console.log("Hello!");
}

greet(); // Hello!
greet(); // Hello!  — call it as often as you like`}
</CodeExample>

Declaring a function does not run it. The body only executes when you *call* it with `greet()`.

## Parameters and arguments

A function becomes useful when it can work on different inputs. *Parameters* are the named inputs in the declaration; *arguments* are the actual values you pass when calling.

<CodeExample language="js" preview>
{`function greet(name) {
  console.log(\`Hello, \${name}!\`);
}

greet("Ada");   // Hello, Ada!
greet("Grace"); // Hello, Grace!`}
</CodeExample>

Here `name` is the parameter; `"Ada"` and `"Grace"` are the arguments. You can take several parameters, separated by commas.

<CodeExample language="js" preview>
{`function describe(name, hobby) {
  console.log(\`\${name} enjoys \${hobby}.\`);
}

describe("Ada", "cycling"); // Ada enjoys cycling.`}
</CodeExample>

## Returning a value

`console.log` only prints. To hand a *result* back so other code can use it, a function must `return` a value.

<CodeExample language="js" preview>
{`function add(a, b) {
  return a + b;
}

const sum = add(3, 4);
console.log(sum);        // 7
console.log(add(10, 5)); // 15`}
</CodeExample>

`return` does two things: it sends the value back to the caller, and it ends the function immediately — any code after a reached `return` does not run. A function with no `return` gives back `undefined`.

A returned value can be used like any other value: stored in a variable, logged, or passed into another function.

<CodeExample language="js" preview>
{`function double(n) {
  return n * 2;
}

console.log(double(double(5))); // double(5) is 10, then double(10) is 20`}
</CodeExample>

## Scope: local vs global

Variables declared **inside** a function are *local* — they exist only while the function runs and cannot be seen from outside. Variables declared outside any function are *global* and visible everywhere.

<CodeExample language="js" preview>
{`const appName = "Calculator"; // global

function showInfo() {
  const version = "1.0"; // local to showInfo
  console.log(\`\${appName} v\${version}\`); // can read the global
}

showInfo();              // Calculator v1.0
console.log(appName);    // Calculator — fine, it's global
// console.log(version); // would error — version is local to showInfo`}
</CodeExample>

Keeping variables local is a feature: each function manages its own values without stepping on the rest of the program. Lean on local variables and avoid piling everything into the global scope.

## Function expressions and arrow functions

A function is also a value, so you can store one in a variable. That is a *function expression*.

<CodeExample language="js" preview>
{`const square = function (n) {
  return n * n;
};

console.log(square(5)); // 25`}
</CodeExample>

Modern JavaScript has a shorter syntax for this called an **arrow function**. It drops the `function` keyword and uses `=>`.

<CodeExample language="js" preview>
{`const square = (n) => {
  return n * n;
};

// For a single return, you can shorten it even more:
const cube = (n) => n * n * n;

console.log(square(5)); // 25
console.log(cube(3));   // 27`}
</CodeExample>

When the body is just one expression, you can drop the braces and `return` — the value is returned automatically. You will see arrow functions everywhere in modern code (and constantly in React), so it is worth recognizing both forms. They do the same job; arrows are just more compact.
