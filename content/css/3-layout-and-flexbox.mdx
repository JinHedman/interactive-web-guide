So far your card stacks everything top to bottom. Real layouts place things side by side, center them, and space them evenly. This chapter covers `display` — the property that controls how an element flows — and **flexbox**, the modern tool for arranging items in a row or column. We will turn the profile card into an avatar beside a block of details.

## display: how an element flows

Every element has a default `display` value that governs its layout behavior.

- **`block`** — takes the full width available and starts on a new line. `<div>`, `<p>`, headings.
- **`inline`** — flows within a line of text and ignores width/height. `<span>`, `<a>`.
- **`inline-block`** — flows inline like text, but *accepts* width, height, and vertical padding.
- **`none`** — removes the element from the page entirely (no space reserved).

<CodeExample preview files={[

<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <p>
      <span class="tag">math</span>
      <span class="tag">writing</span>
      <span class="tag hidden">secret</span>
    </p>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body { font-family: system-ui, sans-serif; padding: 1rem; }
.tag {
  display: inline-block;
  background: navy;
  color: white;
  padding: 4px 10px;
  border-radius: 999px;
  margin-right: 6px;
}
.hidden { display: none; }`
  }
]} />

The third tag has `display: none`, so it takes up no space at all.

## Flexbox: rows and columns made easy

Set `display: flex` on a container and its direct children become **flex items**, laid out in a row by default. Flexbox handles spacing and alignment for you — no floats, no margin tricks.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <div class="card">
      <div class="avatar"></div>
      <div class="details">
        <h1>Ada Lovelace</h1>
        <p>Mathematician and writer.</p>
      </div>
    </div>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body { font-family: system-ui, sans-serif; padding: 1rem; }
.card {
  display: flex;
  gap: 1rem;
  max-width: 420px;
  padding: 1rem;
  background: #f3f4f6;
  border-radius: 8px;
}
.avatar {
  width: 64px;
  height: 64px;
  border-radius: 50%;
  background: navy;
  flex-shrink: 0;
}
.details h1 { font-size: 1.25rem; margin: 0; }
.details p { margin: 0.25rem 0 0; color: #444; }`
  }
]} />

The avatar and the details sit side by side because the `.card` is a flex container. `gap: 1rem` puts space between them without any margins.

## Direction, alignment, and wrapping

A few properties on the flex container do most of the work:

- **`flex-direction`** — `row` (default) or `column` to stack vertically.
- **`justify-content`** — aligns items along the main axis: `flex-start`, `center`, `space-between`, `space-around`.
- **`align-items`** — aligns items along the cross axis: `stretch` (default), `center`, `flex-start`.
- **`gap`** — space between items.
- **`flex-wrap`** — `wrap` lets items drop onto a new line when they run out of room.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <div class="toolbar">
      <strong>Ada Lovelace</strong>
      <nav class="links">
        <a href="#about">About</a>
        <a href="#work">Work</a>
        <a href="#contact">Contact</a>
      </nav>
    </div>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body { font-family: system-ui, sans-serif; padding: 1rem; }
.toolbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 1rem;
  padding: 0.75rem 1rem;
  background: #f3f4f6;
  border-radius: 8px;
}
.links { display: flex; gap: 1rem; }
.links a { color: navy; text-decoration: none; }
strong { font-size: 1.1rem; }`
  }
]} />

`justify-content: space-between` pushes the name to the left and the links to the right; `align-items: center` keeps them vertically centered. This row-with-something-on-each-end pattern is everywhere on the web.

Interactive elements like these links should also react to the pointer and keyboard. A selector with `:hover` (e.g. `.links a:hover { text-decoration: underline; }`) styles an element while the mouse is over it, and `:focus` styles it while it is keyboard-focused — add both so the navigation gives clear feedback to mouse and keyboard users alike.

## A peek at grid

Flexbox excels at laying out items in *one* direction — a row or a column. When you need to control rows **and** columns at the same time — a true two-dimensional layout like a photo gallery or a dashboard — `display: grid` is the better tool.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <div class="gallery">
      <div></div><div></div><div></div>
      <div></div><div></div><div></div>
    </div>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body { font-family: system-ui, sans-serif; padding: 1rem; }
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 0.75rem;
}
.gallery div {
  background: navy;
  aspect-ratio: 1;
  border-radius: 6px;
}`
  }
]} />

`grid-template-columns: repeat(3, 1fr)` makes three equal columns; items flow into rows automatically. You do not need to learn grid in depth now — just recognize the situation: aligning content in two dimensions means grid, while a single row or column means flexbox. The Read more in the next chapter links to a full grid reference.
