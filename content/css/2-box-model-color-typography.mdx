<InShort>Every element is a box of **content → padding → border → margin**; you color it with named/hex/`rgb`/`hsl` values, size it with `px`/`rem`/`em`/`%`, and style its text with a few `font-*` properties.</InShort>

You can target elements now. This chapter gives you the properties that do most of the visual work: spacing through the box model, colors in their common formats, the units you measure with, and the handful of properties that style text. We will keep refining the same profile card.

## Every element is a box

The browser lays out each element as a rectangular box with four layers, from the inside out:

- **Content** — the text or image itself.
- **Padding** — space *inside* the box, between content and border.
- **Border** — a line around the padding.
- **Margin** — space *outside* the box, pushing other elements away.

<CodeExample preview files={[

<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <div class="card">
      <h1>Ada Lovelace</h1>
      <p>I find patterns in numbers and write algorithms for fun.</p>
    </div>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body { font-family: system-ui, sans-serif; padding: 1rem; }
.card {
  background: #f3f4f6;
  padding: 16px;        /* space inside, around the text */
  border: 2px solid navy;
  margin: 24px;         /* space outside, away from edges */
}`
  }
]} />

You can set each side separately (`padding-top`, `margin-left`) or use shorthand. `margin: 24px` sets all four sides; `padding: 8px 16px` sets top/bottom then left/right.

## box-sizing: border-box

<InShort>`box-sizing: border-box` makes `width` include padding and border, so a `300px` box stays `300px` — set it on `*` once at the top of every stylesheet.</InShort>

By default, `width` measures only the content. Padding and border are then *added on top*, so a box you set to `width: 200px` with `20px` of padding actually takes up `240px`. That math is annoying.

<WhyItMatters>
Side-by-side boxes that mysteriously overflow onto a second line are almost always this default biting you. One `* { box-sizing: border-box; }` line makes widths behave the way you expect — which is why nearly every real stylesheet opens with it.
</WhyItMatters>

Set `box-sizing: border-box` and `width` includes padding and border instead — what you ask for is what you get. Apply it to everything once at the top of your stylesheet:

<CodeExample language="css">
{`* {
  box-sizing: border-box;
}

.card {
  width: 300px;     /* the card is exactly 300px wide */
  padding: 16px;    /* padding now fits inside that 300px */
  border: 2px solid navy;
}`}
</CodeExample>

This is one of the first lines almost every stylesheet includes. Use it.

## Colors

CSS accepts colors in several formats. The common ones:

- **Named** — `navy`, `tomato`, `white`. Handy, but limited.
- **Hex** — `#1e3a8a`. Three pairs of hex digits for red, green, blue.
- **rgb** — `rgb(30 58 138)`. The same channels as numbers 0–255.
- **hsl** — `hsl(222 64% 33%)`. Hue, saturation, lightness — easy to tweak by hand.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <div class="card">
      <h1>Ada Lovelace</h1>
      <p>I find patterns in numbers and write algorithms for fun.</p>
      <span class="tag">mathematician</span>
    </div>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body { font-family: system-ui, sans-serif; padding: 1rem; }
.card { background: #f3f4f6; padding: 16px; border-radius: 8px; }
.card h1 { color: hsl(222 64% 33%); }
.card p { color: rgb(68 68 68); }
.tag { color: white; background: tomato; padding: 2px 8px; border-radius: 4px; }`
  }
]} />

Hex is the most common in the wild; `hsl` is the friendliest when you want a slightly lighter or darker shade.

## Units

Numbers in CSS need units. The four you will use most:

- **`px`** — absolute pixels. Predictable; good for borders and fine details.
- **`rem`** — relative to the *root* font size (the `<html>` element, usually 16px). `1.5rem` = 24px. Great for font sizes and spacing because it scales with user settings.
- **`em`** — relative to the *current element's* font size. Useful but compounds when nested.
- **`%`** — relative to the parent. Common for widths in fluid layouts.

<CodeExample language="css">
{`html { font-size: 16px; }   /* the root */

.card {
  width: 80%;               /* 80% of its parent's width */
  padding: 1rem;            /* 16px */
  font-size: 1rem;          /* 16px */
  border: 1px solid navy;   /* a crisp 1px line */
}`}
</CodeExample>

A good default: `rem` for font sizes and spacing, `px` for borders, `%` (or `max-width`, coming in chapter 4) for fluid widths.

## Typography

Four properties cover most text styling:

- **`font-family`** — the typeface, with fallbacks listed in order. End with a generic family like `sans-serif`.
- **`font-size`** — the text size.
- **`font-weight`** — `normal` (400), `bold` (700), or a number.
- **`line-height`** — vertical space between lines. A unitless value like `1.5` means 1.5× the font size and is the recommended way to set it.

<CodeExample preview files={[
  {
    name: "index.html",
    lang: "html",
    code: `<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <div class="card">
      <h1>Ada Lovelace</h1>
      <p>I find patterns in numbers and write algorithms for fun. Mathematician, writer, and arguably the first programmer.</p>
    </div>
  </body>
</html>`
  },
  {
    name: "styles.css",
    lang: "css",
    code: `body {
  font-family: "Segoe UI", system-ui, sans-serif;
  padding: 1rem;
}
.card { background: #f3f4f6; padding: 1.5rem; border-radius: 8px; max-width: 360px; }
.card h1 { font-size: 1.75rem; font-weight: 700; margin: 0 0 0.5rem; }
.card p { font-size: 1rem; line-height: 1.6; color: #444; margin: 0; }`
  }
]} />

Starting `font-family` with `system-ui` uses the reader's native operating-system font — fast and familiar, with no download.
